Lecture Notes - HTML

Key terms

HTML
HyperText Markup Language - a standardized way of combining plain content (e.g. text) with markup, information meant to describe the plain content's role and appearance as opposed to being displayed literally.
Tag
A kind of markup used in HTML. At a minimum, a tag consists of "angle brackets" (< and >) surrounding an alphanumeric name. A tag tells the browser to create a corresponding HTML element whose name matches the specified type.
Element
An object created by the browser to correspond to an HTML tag.
Attribute
An optional property of an element.
Node
An overarching term that means an element, comment, or plain text inside a HTML document.

For more information about these and other terms related to HTML, see The MDN Glossary. If you want really detailed information about how exactly HTML works, see the HTML Living Standard, available here.

Document Structure

An HTML document consists of a set of nested nodes which form a tree structure. Just like in Data Structures, the document contains a root element with zero or more child nodes. In a HTML document tree, a node can be an element, comment, or plain text. Comments and plain text can only be leaf nodes, while elements can either be leaves or intermediate nodes.

Examples

A sub-tree where a paragraph (<p>) element has one text node child: 

    <p>
        This is a paragraph with nothing but a single text node.
    </p>

A sub-tree where an unordered list (<ul>) has two list item (<li>) children, each of which have a text node: 

    <ul>
        <li>a list item with a text node</li>
        <li>another list item with a text node</li>
    </ul>

A sub-tree where a paragraph has a mix of text and elements as children. In this case, the text is split into two separate nodes by the element between them. 

    <p>
        a text node <i>and</i> a second text node
    </p>

Inclusion restrictions

Not all kinds of elements are allowed to appear in all parts of an HTML document tree. For example:

  • The root element must be <html>.
  • <html> can only have one <head> and one <body> as children.
  • Some elements can only appear in <head>, such as <title>. Others can only appear in <body>. A few, such as <script>, can appear in either.
  • Some elements can only have a single text child, such as <title>.
  • Some elements cannot have any children, such as <meta>.
  • Some elements are allowed to have only specific types of elements as children.

Browsers are sometimes more flexible than the rules defined in the HTML standard. For example, the previous paragraph contains an unordered list, even though the paragrapgh element is only supposed to allow phrasing content (see the specification for the <p> element). Just like how most people will understand if you say something close to but not "proper" English (e.g. using "me" in place of "I" or vice-versa), browsers will understand if you do something close to but not technically correct. However, be aware that some incorrect element combinations will result in the browser interpreting your HTML in unexpected ways or ignoring those elements entirely.

Note: comments are allowed inside pretty much any other element that can have child nodes.

HTML elements

A HTML element has three components - type, attributes, and children. A webpage will represent an element using a tag, as described under Key Terms. There are two patterns for writing tags - either a combination of an open tag and close tag, or with a self-closing tag. The examples above all show open + close tags, whereas something like <meta charset="UTF-8"/> is an example of a self-closing tag. The type and attributes will always appear inside an open tag or self-closing tag. Children appear between open and close tags. Self-closing tags cannot contain children. Outside of a few exceptions, you will always use open + close for elements with children and self-closing for elements without.

Attributes

Attributes are like member variables or a map/dictionary that contains properties of an element. Attributes let us specify behavior and/or visual aspects of HTML elements. The links contained in this page are an example of specifying behavior - each <a> tag contains an attribute called "href" that says where the browser should navigate to when the user clicks on the link. Other attributes might affect both function and appearance, such as the "disabled" attribute you can add to a button.

Attributes all have a name, and most match a name to a value. It's permissible to specify just a name, in which case the attribute automatically uses an empty string as its value. This is useful for some attributes where the browser does something based on the presence of the attribute alone. For example, the following behave the same, but the first one is more compact:

<button disabled>Don't click me</button>
<button disabled="true">Don't click me</button>

Attributes for which you want to specify a non-default value have an equals sign (=) followed by the value, optionally in quotes. While it's valid to say width=80 or align=left, any value with whitespace or characters that the browser might interpret as part of the tag (", ', <, etc.) won't work correctly without being enclosed in quotes. I recommend using double quotes ("") as a default, even for numbers and booleans. Browsers will still know the target data type and consistency makes it easier to errors in the event you do need quotes.

Element types

The following are just a few examples of the many types of elements you can include in an HTML document.

span
Span
inline

Can contain any other element that goes in the document body. Adds no layout or styling whatsoever.

div
Division
block

Can contain any other element that goes in the document body. Puts the content on its own line.

p
Paragraph
block

Encloses a paragraph of text and phrasing content (e.g. <b>). Adds spacing above and below the text, but not in between each line.

ul
Unordered list
block

Used to create lists with bullets (dots) for each item. Contains <li> elements.

ol
Ordered list
block

Used to create lists with sequential numbers or letters for each item. Contains <li> elements.

li
List item
block

Used to identify a single item in a list. Can appear in <ul> or <ol>

code
Code
inline

Used to represent a snippet of code. Uses a monospace font.

pre
Preformatted
block

Used to represent a snippet of code. Uses a monospace font. Preserves any whitespace such as line breaks.

img
Image
inline

Displays an image using the image data located at the URL in the "src" attribute.

a
Anchor
inline

When clicked, tells the browser to navigate to the URL in the "src" attribute. Will turn blue if the user has not visited the destination before, purple if they have.

Miscellaneous

Common inclusions

There are a few elements and attributes that it is common to include for clarity to either the user or their software:

<!DOCTYPE html> as the very first line of the webpage
Tells the browser that it's reading HTML, as opposed to plain text, an image, etc.
lang="en" (or whatever language code is appropriate) in <html>
Tells screen readers what language they should read the contents of the document in.
<meta charset="UTF-8"> inside of <head>
Tells the browser that the document uses the UTF-8 character set (as opposed to ASCII or any other language-limited character set). I always recommend declaring and using UTF-8 both so you can handle multiple human languages in your website and for special symbols like math operators or emoji ⇒ 😎

Character references

In other languages you may be used to the idea of escaping control characters. For example, if you have a Java or C++ string that contains a double quote (e.g. "\"free\" pizza"), you would need to put a backslash before the quote is part of the string, not the end of the string. The same concept applies to HTML - there are some characters like < that you need to escape so that the browser knows you want to display a < rather than starting a tag.

The HTML way of escaping characters is to use some short name for the character between an ampersand (&) and semicolon (;). If you look at the source code for this page, you'll see a lot of these scattered throughout. Even in this paragraph, I had to type &amp; in order to make the ampersand appear.

More generally, you can use this technique to refer to any character. While some characters have special names (amp, lt, gt, quot, apos), you can always refer to a character using its Unicode number. For example, &#x1F642; will translate to 🙂 based on the #x indicating that the following is a hexidecimal number and 1F642 being the Unicode code for slightly smiling face. Of course, if you declare your charset to be UTF-8 you can type/paste the equivalent in directly 🙂
