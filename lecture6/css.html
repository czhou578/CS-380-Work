CSS Lecture Notes

CSS Overview

CSS stands for Cascading Style Sheets. It was invented as a compliment to HTML so that authors could customize the appearance of their webpages and make that appearance consistent across different browsers.

CSS is a declarative language, meaning that instead of telling the browser exactly what to do (which we do in imperative languages like JavaScript) we give it rules to follow and it figures out the outcome.

CSS Syntax

CSS generally takes the following form: 

selector {
    property: value;
}
selector
Tells the browser which elements we want to change the style of.
property
Tells the browser what quality of that element we want to change, such as text color, background color, how much space to put around the element, etc.
value
Tells the browser what we want to set the property to, such as a specific color, a certain amount of space, a certain font style, etc.
You can combine multiple property: value; rules in one set of braces ({}) and they will all apply to the elements targeted by the selector.

Common Properties

There are a huge number of CSS properties, many of which have niche uses or specific elements to which they apply. For a comprehensive list, see the MDN reference. Below are a list of common or particularly useful properties you should know.

Colors

There are multiple ways to specify values for color properties. It's common to see #RGB or #RRGGBB values specified on websites that list color values, but some of the alternatives may make sense for specific use cases.

Property formats

  1. Six hexidecimal digits, such as #003D79 (the nav bar background color). Each pair of digits represents how much red, green, and blue to include, respectively.
  2. Three hexidecimal digits, such as #0F0 (pure green). This is easy-to-use shorthand for six digits; #FAB becomes #FFAABB.
  3. A color name, such as black. Browsers have a wide but limited number of color names, so make sure you use a name that they all recognize.
  4. The rgb() function. This takes three arguments that represent red, green, and blue as integers between 0 and 255. Works like six digit hexidecimal format.
  5. The rgba() function. This works just like rgb() except that it takes a fourth argument, called the alpha value. Alpha can be a decimal value between 0 and 1.0. It represents how transparent the color should be, i.e. how much it blends with the color behind it.

Uses

  • background/background-color
  • color (used for text)
  • border/border-color

Text

font-size
Used to set how large the text is.
font-weight
Can be used to set how thick the letters in the text are. Often used to make bold text, but some fonts have a range of thicknesses, represented in multiples of 100 from 100 to 800 (normal typically being 400).
text-decoration
Allows you to add lines above or below text. Can be used for simple underlines or something fancy like spelling errors.
font-style
Allows you to make text italic.
text-align
For block elements, defines how to place the text within the block. You can use left, right, or center to place spare space on each line to the right, the left, or evenly on both sides, respectively. Note: if you might end up handling languages that read from right to left, you may want to consider the difference between values such as left which forces left alignment vs. values such as start which forces alignment to the natural starting place for reading.

Selectors

Selectors are how you define which elements should receive a set of style rules.

Selector Types

There are a few different types of CSS selectors:

Type
Type selectors use the name of a HTML tag to apply a style to all tags with that name. For example, h1 { color: 'darkgray'; } will apply the color to all <h1> elements.
Class
You can add the "class" attribute to any HTML element as another way to select elements with CSS. In your CSS, you add a dot/period before the name of the class. Example: if you have an element <div class="important"> and some CSS .important { font-weight: bold; } then the div will have bold text.
Id
Id selectors work just like classes, except that you are only supposed to have one element in your entire page with a given id, whereas you can apply a class to as many elements as you like. In CSS, an id selector uses the hash/pound symbol (#) instead of a dot/period.
Pseudo-element
The browser allows you to create some elements purely out of CSS. These are called pseudo-elements. For example, if you want to add some extra styling or content before or after an exisitng element, you can use :before or :after. These aren't particularly common, but you can use them to add extra characters, symbols, or images before or after elements you defined in your HTML.
Pseudo-class
While you have to apply classes directly to elements using the "class" HTML attribute, there are some classes that the browser automatically applies to some elements. These are called pseudo-classes. Examples include :hover for when the user's cursor is over the element or :nth-child() to select a specific element or subset of elements within the same parent element. Pseudo-classes always begin with a colon (:).
Attribute
You can also select elements by specifying that they must have an attribute with a specific value. For example, input[type="text"] will only apply to input elements with a HTML attribute of "type" that's equal to "text" (<input type="text"> but not <input type="checkbox"> or <input>). There are several variations that let you specify a prefix, suffix, regular expression, or other attributes. Attribute selectors are surrounded by square brackets ([]).

Combining Selectors

You can use mutliple selectors together to make more specific selectors. You can combine multiple of the same type or mix different types. Here are the different ways in which you can combine them:

Adjacent
Putting two selectors together with nothing between them will act like a logic AND. For example, div.important will only apply to elements with a type of div and a class of "important". You can can combine multiples of any selector type this way except for type (an element cannot have more than one type). Example: a#foo.bar.baz:visited[href^="http"].
Space-separated
Adding whitespace between selectors says that you're looking for an element that's a descendant of another element. For example, p a will apply to any a elements that are children, grandchildren, etc. of a p. You can chain multiple selectors with spaces, e.g. body #bonus article .info.
Greater than (>)
Adding a greater than symbol between selectors (with or without whitespace) says that you're looking for an element that's a direct child of another element. For example, ul.small > li will apply to any list items directly inside unordered lists with the "small" class, but not to list items that are in sub-lists.
Plus (+) and Tilde (~)
Work like greater than, but between siblings, or in other words two children of the same element. The plus sign means that the browser should only look for a elements that are directly next to each other, whereas the tilde means that the second element just has to come after the first element (there can be others in between them). For example, .main + .alt will apply to any elements with the class "alt" that are immediately after an element with the class "main" within the same parent, while .main ~ .alt will apply to any elements with the class "alt" that are somewhere after an element with the class "main" within the same parent. In the following HTML, the plus operator applies to just the first "alt", while the tilde applies to all following "alt"s. 
<div>
    <div class="main">Elizabeth</div>
    <div class="alt">Liz</div>
    <div class="alt>Beth</div>
</div>
Note: there are no ways to select a parent or a previous sibling, only ways to go down or forwards in the tree of elements.

Specificity

In the case of conflicting styles, browser will prefer whichever style came from the more specific selector. The specificity of a selector is a score that the browser gives based on the parts that make up the selector. You can think of it like a comparator or a key function that you would use to sort a list in Java, Python, etc.

Here are the rules for which style the browser selects:

  1. Any style defined on the HTML element itself using the style HTML attribute.
  2. Any style rule with !important on it. Note that this applies to each property, not a selector: 
    .my-style {
    padding: 4px;
    color: green !important;
}
  3. Any style that comes from a selector with an id in it. More ids take priority over fewer ids, although there's normally no reason for more than one id.
  4. If two selectors have the same number of ids, then whichever one has more classes, pseudo-classes*, or attribute selectors. One of each would win over two classes, but two classes and two attribute selectors would win over one of each or three classes or two classes and one pseudo-class, etc.
  5. If two selectors have the same number of ids and classes, then whichever one has more types or pseudo-elements. For example, ul > li will win over li.
  6. If two selectors have the same number of ids, classes, and types, then whichever one was read last by the browser. The browser will always load its own default stylesheet first, which is why even if you use a selector like a which would tie the default, yours loads second so it overwrites the default.
*The :not() pseudo-class does not add to the specificity score. li:first-child and li:not(:first-child) have the exact same specificity score.

Size measurements

CSS provides several ways to specify size values:

px
Pixels. While this used to mean the smallest individual changeable part of the monitor, small devices will underreport the number of pixels in their display so things don't appear tiny on them. While your phone may actually have a higher resolution than your laptop, it will typically report a much lower width or height in pixels. Regardless, px is a common default unit of measurement for sizes.
%
Percent. This defines the width of the element relative to the size of the nearest block element ancestor.
em
Represents the size of one capital letter M in the element's font size.
vw/vh
Viewport width/height. Similar to percent, but relative to the viewport, i.e. the whole visible portion of the page.

There are several other measurements that refer to physical sizes, such as cm for centimeters, in for inches, or pt for points (the sizing used in Mircosoft Word). These measurements are typically used for printable material; for web pages we use pixels instead.

The Box Model

When rendering HTML elements onto the screen, the browser surrounds each element with a rectangle/box. The rectangle allows the browser to make sure that elements don't overlap (unless we tell it to do that explcitly). It also lets us customize the size of the element and the space around it. The logic and the CSS properties that govern these boxes are collectively known as the box model.

Note: the box model only applies to block elements, such as <div> or elements with a CSS display property that makes it a block (more on that in a later section). Any inline elements will instead contribute to the box belonging to their nearest block ancestor.

Box Model Layers

In order to give us more fine-grained control over spacing, the box around each element has several layers that we can size individually. I call these layers because each successive one surrounds the prevous. Going from inside to outside, the layers are:

  1. Content
  2. Padding
  3. Border
  4. Margin

Content is the innermost layer. The content box will default to fitting the text, images, inline children, etc. contained within the element. However, you can use the CSS width and height properties to set the size of the content box (unless you set the CSS property background-clip; see below).

Padding surrounds the content. It allows you to add space between the content and the border. By default, padding will have the same background as the content. For an example, see the bottom of this page ‐ there's 64px worth of space with the <main>'s background color below where the content ends.

The border goes around the padding. Borders are essentially a line that the browser draws around the padding box. You can specify how wide the line is, what drawing style the browser should use (solid, dashed, dotted, etc.), and what color to draw it with. There are separate CSS properties for each of these, or the combined border property that sets all three at once.

The border is where the background of the element ends. Even if you don't specify a border directly, the browser will act as if you have a 0px wide border surrounding the padding. This is relevant when you specify the border-radius property. Even if you don't give an element a border, the border radius will still round the corners of the background.

Finally, margin goes on the outside of the border. It works much like padding except that it does not include the element's background.

Blocks and the Display Property

As noted above, the box model only applies to block-level elements. Some elements such as <div> will default to being block-level, while other elements will default to being inline, such as <span>. You can override this behavior using the CSS property display. In addition to choosing block or inline behavior, there's an in-between value of inline-block. The differences are:

  • block will separate the element from the previous and following elements by putting it on its own line. The browser will apply any box model properties to the element.
  • inline will put the element on the same line as the previous and following elements. The browser will ignore any box model properties of inline elements (width, padding, etc.).
  • inline-block will put the element on the same line as the previous and following elements like it does with inline, but the browser will apply any box model properties like it does with block.

Box Model Modifiers

There are several properties that can override the default behavior of the box model as described above. The most common one I've seen is box-sizing, which changes which layer of the box model is affected by width and height. border-box-example allows you to choose between the default of content-box and the alternative of border-box, the latter of which will make the size properties affect the content, padding, and border combined.

The other notable change you can make to the box model rules is with background-clip. This property changes how far the background extends. By default, it will extend to the edge of the border, which means that if you have a transparent border with > 0 width (example: border: 2px solid transparent;) then the background will extend that width. By changing the background clip you can set it to only match the padding or content box.
